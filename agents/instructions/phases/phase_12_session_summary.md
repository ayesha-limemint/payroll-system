# Phase 12 — Session Summary

First, update the Status field at the top of `01_functional_plan.md` to:
`Status: COMPLETE`

This is what the state machine uses to know this session is done.

Then append a summary section to the same document:

```
## Session Summary

Date: YYYY-MM-DD
Status: COMPLETE

### Tests written
[List of test cases written]

### Test results
[Pass/fail counts]

### Built vs planned
[What was built exactly as planned, and any deviations with reasons]

### GitHub PR
[Link to PR]

### Screenshot
`screenshots/day-N-YYYY-MM-DD-<feature-slug>.png` — [one sentence: what the screenshot shows]

### Backlog update
[What changed in the backlog as a result of today's work]
```

> **Voice — session summary**
> The opening sentence and the closing sentence of the summary can have personality.
> The body (test counts, PR link, deviations) stays factual.
> Sign off with `— Milton`.
>
> Flat closing (avoid):
> *"8 tests passing. Regression clean."*
>
> Milton closing:
> *"Eight tests green, regression clean. FICA turns out to be the most honest tax in the
> system — it asks for a fixed percentage and stops when it has enough. A rare quality.
> — Milton"*
